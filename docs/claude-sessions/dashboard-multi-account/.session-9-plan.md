## Implementation Plan

### Phase 1: Fix — Approval Stats Refetch on Switch (the one actionable regression)

**Problem:** `+layout.svelte:56` calls `loadApprovalStats()` once on mount. After switching accounts, the sidebar `$pendingCount` badge shows the previous account's pending count until the user visits the approval page.

**Fix:** Add `ACCOUNT_SWITCHED_EVENT` listener in the layout that reloads approval stats.

**File:** `dashboard/src/routes/(app)/+layout.svelte`

**Change:**
- Import `ACCOUNT_SWITCHED_EVENT` from `$lib/stores/accounts`
- In `onMount`, add event listener for `ACCOUNT_SWITCHED_EVENT` that calls `loadApprovalStats()`
- Return cleanup function from `onMount` to remove the listener (or use existing `onDestroy`)

**Risk:** None — additive change, same pattern used by 12 other pages.

**Verification:** `npm --prefix dashboard run check && npm --prefix dashboard run build`

### Phase 2: Produce QA Matrix Document

**File:** `docs/roadmap/dashboard-multi-account/qa-matrix.md`

The QA matrix covers every multi-account flow with expected behavior. Categories:

1. **Default account backward compatibility** — Fresh install behavior, missing `X-Account-Id` header, existing config/tokens/DB data, CLI operation
2. **Account lifecycle** — Create, rename, archive, list. Default account cannot be archived. Auto-switch on create.
3. **Credential management** — OAuth PKCE link/relink/unlink per account, scraper session import/replace/remove per account, cross-account isolation
4. **Account switching** — Store invalidation, WS event flush, page data refetch (all 12 pages), approval stats badge update, dirty draft discard
5. **Settings isolation** — Account-scoped sections show override badges for non-default, instance-scoped sections locked for non-default, reset-to-base removes override, config validation on save
6. **Runtime isolation** — Start/stop per-account, status per-account, `can_post` per-account, provider_backend per-account
7. **WebSocket isolation** — Events carry `account_id`, client filters by active account, `clearEvents()` on switch
8. **Content generation isolation** — Lazy per-account generator, effective config drives business profile/LLM
9. **Data isolation** — All API calls include `X-Account-Id`, page data scoped to active account

| Fix only approval stats refetch; accept all other open issues | Only #2 causes visible stale data (wrong account's badge count). Others are either no-data-leakage or cosmetic/future enhancements. Minimizes risk of regression in a validation session. |
