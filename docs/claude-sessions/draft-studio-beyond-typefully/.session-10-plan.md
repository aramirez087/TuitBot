# Session 10 Implementation Plan — Entrypoints & Rollout

## Summary

Session 10 eliminates the remaining split-brain compose flows so every writing action flows through Draft Studio. The home page stops embedding a standalone `ComposeWorkspace` and instead routes into `/drafts`. The calendar "Compose" button creates a server-backed draft and redirects to Draft Studio. Navigation labels, keyboard shortcuts, settings, and docs are updated to reflect one canonical writing system.

## Design Decisions

### D1: Home page becomes a routing surface, not a compose surface

**Options considered:**
- A) Redirect `/` → `/drafts` always → breaks analytics-home users, loses the "landing page" concept
- B) Keep home as a toggle surface but replace composer with a "quick start" card that links to Draft Studio → preserves settings, clean transition
- C) Change `HomeSurface` type to `'drafts' | 'analytics'` and redirect `'drafts'` to `/drafts` → simplest but flash of redirect

**Decision: Option B.** The home page keeps the `homeSurface` toggle but the "composer" option becomes "drafts" — rendering a welcome/quick-start card with:
- "New Draft" button → calls `draftStudio.createDraft()` then `goto('/drafts')`
- "Resume Last Draft" button (if a recent draft exists) → `goto('/drafts?id=${lastDraftId}')`
- Recent drafts list (3-5 items) for quick access
- This avoids a jarring redirect, keeps the analytics option working, and gives a clear entry point

**Rationale:** The charter says "Home becomes a shortcut into Draft Studio, not a parallel composer." A quick-start card is a shortcut. A redirect would be disorienting.

### D2: Calendar compose creates a draft and redirects

**Decision:** The calendar "Compose" button and time-slot click will:
1. Call `api.draftStudio.create({ content_type: 'tweet', content: '', scheduled_for: prefillTime })` to create a server-backed draft
2. Navigate to `/drafts?id=${newId}` where the user writes in Draft Studio
3. Remove `ComposeModal` import from the calendar page entirely

**Tradeoff:** This is a UX change — users currently get an inline modal for quick scheduling. The redirect is slightly more friction but ensures every draft goes through the canonical flow with autosave, revision history, and the full keyboard-first experience. The `scheduled_for` prefill means the time slot context is preserved.

### D3: `Cmd+N` targets Draft Studio

**Decision:** Change `Cmd+N` handler in `+layout.svelte` to:
- Navigate to `/drafts` and trigger new draft creation via a URL param (`/drafts?new=true`)
- `DraftStudioShell` reads this param on mount and calls `createDraft()` if present

### D4: Telemetry via `console.info` structured events

**Decision:** Add lightweight `console.info` calls with structured JSON payloads at key lifecycle points. No external telemetry service — just structured logs that Tauri's devtools / browser console can capture. Format: `console.info('[draft-studio]', { event, ...data })`.

Events to log:
- `draft_created` — source (home-quickstart, calendar-slot, cmd-n, rail-button)
- `draft_selected` — id, source (rail-click, url-deeplink, home-resume)
- `save_failed` — id, error, syncStatus
- `restore_executed` — id, revisionId
- `transition` — id, from, to (draft→scheduled, scheduled→draft, etc.)

### D5: Keep calendar `ComposeModal` as deferred cleanup (Session 11)

**Revised decision:** After further analysis, removing `ComposeModal` from calendar entirely in this session is risky — it affects the calendar week/month view components that emit slot-click events. Instead:
- Change the calendar "Compose" button to create a draft + redirect
- Change time-slot clicks to create a draft + redirect
- Leave `ComposeModal` import in place but unused (dead code) — clean up in Session 11

**Actually, simpler approach:** Just change `openCompose()` to create a draft and redirect instead of setting modal state. The modal never opens. Remove it in the same session since it becomes unreachable code.

## New File: `DraftStudioQuickStart.svelte`

**Path:** `dashboard/src/lib/components/home/DraftStudioQuickStart.svelte`

**Purpose:** Replaces the home-page ComposeWorkspace with a lightweight entry card.

**Structure (~80-120 lines):**
```
<script>
  - Import api, goto, PenLine, Clock, FileText icons
  - State: recentDrafts (3 items), loading
  - onMount: fetch recent drafts via api.draftStudio.list({ limit: 5, status: 'draft' })
  - handleNewDraft(): create draft, goto /drafts?id={id}
  - handleResume(id): goto /drafts?id={id}
</script>

<div class="quickstart">
  <div class="quickstart-header">
    <h2>Draft Studio</h2>
    <p>Your writing workspace</p>
  </div>

  <button class="new-draft-btn" onclick={handleNewDraft}>
    <PenLine /> New Draft
  </button>

  {#if recentDrafts.length > 0}
    <div class="recent-drafts">
      <h3>Recent Drafts</h3>
      {#each recentDrafts as draft}
        <button onclick={() => handleResume(draft.id)}>
          {draft.title || draft.content_preview || 'Untitled'}
          <span class="meta">{relativeTime(draft.updated_at)}</span>
        </button>
      {/each}
    </div>
  {/if}

  <a href="/drafts" class="studio-link">Open Draft Studio →</a>
</div>
```

**Rationale:** Keep it simple. This is a launch pad, not a feature surface.

## Risks

| Risk | Likelihood | Impact | Mitigation |
|------|-----------|--------|------------|
| Calendar redirect breaks quick-compose UX for power users | Medium | Medium | The redirect preserves time-slot context via `scheduled_for` prefill. Users land in Draft Studio with the time already set. Document in rollout notes. |
| `homeSurface` migration: existing users with `'composer'` stored | High | Low | Explicit migration in `loadHomeSurface()` — if persisted value is `'composer'`, map to `'drafts'` and re-persist. |
| `Cmd+N` with `?new=true` creates orphan drafts if user immediately navigates away | Low | Low | Drafts are cheap (soft-deletable). Archive cleanup can handle this. Same behavior as clicking "New" in the rail. |
| Removing ComposeModal from calendar may break onboarding redirect (`?compose=true`) | Medium | Low | The onboarding redirect at line 86 (`$page.url.searchParams.get('compose') === 'true'`) currently opens the modal. Change this to create a draft and redirect to `/drafts?id=${id}` with `scheduled_for` set to today. |
| `DraftStudioQuickStart` needs API call to list recent drafts — adds latency to home page | Low | Low | The list call is lightweight (5 items, summary only). Show skeleton while loading. |
| `docs/composer-mode.md` is 504 lines — large doc to update | Low | Low | Surgical edits to specific sections, not a rewrite. Most content (thread composer, keyboard shortcuts, API endpoints) is still accurate. |
