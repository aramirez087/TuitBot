### Critical Blocker: `$derived` Export Build Failure

`dashboard/src/lib/stores/draftStudio.svelte.ts` lines 74â€“109 export five `$derived` values (`activeDrafts`, `scheduledDrafts`, `postedDrafts`, `currentTabDrafts`, `selectedDraft`, `tabCounts`). **Fix:** Convert each exported `$derived` to a getter function that returns the derived value, keeping reactivity internal. This pattern matches the existing getter functions at lines 115â€“161.

## Task Breakdown and Order of Operations

### Phase 1: Fix Build Blocker (must be first)

#### Task 1.1: Fix `$derived` export in `draftStudio.svelte.ts`

**File:** `dashboard/src/lib/stores/draftStudio.svelte.ts`

**Problem:** Svelte 5 cannot export `$derived()` from a `.svelte.ts` module file. The build fails at line 74.

**Approach:** Convert the six exported `$derived` bindings to non-exported `$derived` locals + exported getter functions. This preserves reactivity within the module while exposing values through function calls.

```ts
// BEFORE (broken):
export const activeDrafts = $derived(...);

// AFTER (working):
const activeDrafts = $derived(...);
export function getActiveDrafts() { return activeDrafts; }
```

Apply to: `activeDrafts`, `scheduledDrafts`, `postedDrafts`, `currentTabDrafts`, `selectedDraft`, `tabCounts`.

**Consumers to update:** Every `.svelte` file that imports these named exports must switch to calling the getter. Search for:
- `import { activeDrafts` / `activeDrafts,` / etc.
- `import { selectedDraft` / `selectedDraft,`
- `import { currentTabDrafts`
- `import { tabCounts`
- `import { scheduledDrafts`
- `import { postedDrafts`

Each consumer wraps the call in `$derived()` locally:
```svelte
// BEFORE:
import { activeDrafts } from '$lib/stores/draftStudio.svelte';
// used as: {activeDrafts.length}

// AFTER:
import { getActiveDrafts } from '$lib/stores/draftStudio.svelte';
const activeDrafts = $derived(getActiveDrafts());
// used as: {activeDrafts.length}
```

**Files likely affected (grep for imports):**

- `DraftRail.svelte` (or equivalent rail component)

### Phase 2: Handle `prefill_schedule` URL Param

#### Task 2.1: Parse and consume `prefill_schedule` in DraftStudioShell

**File:** `dashboard/src/lib/components/drafts/DraftStudioShell.svelte`

**Problem:** Calendar creates drafts and navigates to `/drafts?id={id}&prefill_schedule={iso}` but the Shell ignores `prefill_schedule`. Users who click a time slot on the calendar land in Draft Studio without the time pre-filled.

**Approach:**
1. In the Shell's `onMount` or URL-parsing block, read `prefill_schedule` from `URLSearchParams`.
2. Pass it as a prop to `DraftDetailsPanel` (e.g., `prefillSchedule={prefillScheduleParam}`).
3. In DraftDetailsPanel, if `prefillSchedule` is provided and the draft is not already scheduled, auto-populate `scheduleDate` and `scheduleTime` from the ISO string, and optionally auto-open the schedule section.
4. Clean the URL param after consumption (same pattern as `?new=true` cleanup).

**Design decision:** Do NOT auto-submit the schedule. Pre-populate the pickers and let the user confirm. This avoids surprise scheduling.

**Files modified:**
- `DraftStudioShell.svelte` â€” parse param, pass prop, clean URL
- `DraftDetailsPanel.svelte` â€” accept `prefillSchedule` prop, populate date/time fields in `$effect`

**Verification:** Manual test: navigate to `/drafts?id=1&prefill_schedule=2026-03-10T14:00:00Z`, confirm date/time pickers are pre-populated.

## Key Design Decisions

| Decision | Rationale |
|----------|-----------|
| **Getter functions for `$derived` exports** | Svelte 5 forbids exporting `$derived` from modules. Getters are the idiomatic workaround and match the existing pattern in the same file (lines 115â€“161). Consumers use `$derived(getX())` locally for reactivity. |
| **Pre-populate schedule pickers, don't auto-submit** | Users clicking a calendar slot expect their time to be remembered, but auto-scheduling without confirmation would be surprising. Pre-populating respects intent while preserving control. |
| **Delete dead components outright** | `ComposeModal`, `ComposerPromptCard`, `ComposerTipsTray`, `ComposerShortcutBar` have zero remaining consumers after Session 10 changes. No deprecation period needed for internal components. |
| **Extract 5 sections from DraftDetailsPanel** | Brings the file from 935 to ~400 lines (CLAUDE.md limit). Each section is self-contained with clear props/events interface. Collocated in same directory. |
| **Ship recommendation contingent on build fix** | The `$derived` export bug is a production-blocking regression. No ship without it. |
