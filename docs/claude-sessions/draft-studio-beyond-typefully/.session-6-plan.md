# Session 6 — Rail Keyboard Navigation & Multi-Draft Actions

## Goal

Make the draft rail faster than Typefully by supporting dense scanning, keyboard navigation, and zero-confusion quick actions. A user should be able to manage drafts without the mouse, destructive actions must be explicit, and rail counts/filters must stay in sync with tab and store state.

## Design Decisions

### 1. Roving tabindex on rail items

Use [roving tabindex](https://www.w3.org/WAI/ARIA/apg/patterns/listbox/) on rail entries: the focused item gets `tabindex="0"`, all others get `tabindex="-1"`. The rail container gets `role="listbox"` and each item `role="option"`. This is the WAI-ARIA listbox pattern — screen readers announce the list and position.

**Why not `aria-activedescendant`?** That pattern requires the container to hold focus, which makes `Tab` to escape the rail ambiguous. With roving tabindex, `Tab` naturally moves focus to the composer zone.

### 2. Focused index lives in DraftRail, not the store

The focused index is ephemeral UI state (like which list item has keyboard focus). It doesn't need to survive page navigation or be shared across components. Keeping it in the rail component avoids store coupling and keeps the store focused on data.

The store's `selectedId` (which draft is open in the composer) is separate from the rail's `focusedIndex` (which item has keyboard focus). Arrow keys move the focus ring; `Enter` syncs focused → selected.

### 3. DraftRailItem replaces DraftRailEntry

Renaming to `DraftRailItem` (matching the deliverable name) and adding:
- Quick action buttons (archive, duplicate, restore) that appear on hover/focus.
- Focus ring via global `:focus-visible` (already in `app.css`).
- `tabindex` prop controlled by parent.
- Callback props for actions.

The old `DraftRailEntry.svelte` is deleted. Only `DraftRail.svelte` and `DraftStudioShell.svelte` import it.

### 4. Undo buffer for archive (toast-based)

When a draft is archived, show a transient toast/banner at the bottom of the rail: "Draft archived — Undo". The undo calls `restoreDraft(id)`. The toast auto-dismisses after 5 seconds. This is the "undo-safe path for destructive behavior" from the session spec.

Implementation: A `lastArchived` state in the rail component holding `{ id, title, timeout }`. On archive, set it; on undo or timeout, clear it. No global toast system needed — the rail is the only consumer.

### 5. Command palette gets a "Draft Studio" category

Add a new category to the palette: `'DraftStudio'`. Actions:
- New draft (`N`)
- Duplicate current draft (`Cmd+D` in rail context)
- Archive current draft (`Backspace`/`Delete` in rail context)
- Restore archived draft (`R` in archive tab)
- Jump to rail (`Escape` from composer)

The palette already supports `when` filtering. Add `when: 'draft-studio'` for draft actions and pass a new `context` prop to the palette (or always show them when in draft studio).

**Decision**: Since the command palette is rendered inside `ComposeWorkspace`, not at the shell level, draft actions need to either: (a) be injected via a prop, or (b) the palette scope needs to be lifted. **Plan**: Add an `extraActions` prop to `CommandPalette` that the shell passes down through the workspace. This keeps the palette generic while allowing draft-specific actions. The `onaction` callback already exists and can handle the new IDs.

Actually, re-reading the code — `CommandPalette` is rendered inside `ComposeWorkspace`. The shell controls the workspace via props. The cleanest approach: add a new `commandPaletteActions` prop to `ComposeWorkspace` that injects extra actions into the palette. The workspace passes them through. The shell provides the draft-specific actions list.

| Context | Key | Action |
|---------|-----|--------|

### 7. No bulk archive in this session

The session instructions mention "bulk archive" but the deliverables don't list a multi-select UI. Multi-select (checkboxes, shift-click ranges) is significant UX complexity. **Decision**: Defer bulk archive to Session 07 (metadata & filters) or a dedicated session. Session 6 focuses on single-item keyboard actions which already cover the 80% case. Document this deferral in the handoff.

**Template structure**:
```html
<div role="option" aria-selected={selected} tabindex={tabindex} class:focused class:selected>

    <span class="type-badge">...</span>
```

**Styling**:
- Actions container: `opacity: 0` by default, `opacity: 1` on parent `:hover` or `.focused`.
- Focus ring: rely on global `:focus-visible` from `app.css`.
- Prevent action button clicks from bubbling to the item's select handler.

### Step 4: Rewrite DraftRail.svelte with keyboard navigation

**What**: Add roving tabindex, `keydown` handler, undo toast, tab switching via number keys, and `Posted` tab.

**New state**:
```ts
let focusedIndex = $state(0);
let lastArchived = $state<{ id: number; title: string; timeout: ReturnType<typeof setTimeout> } | null>(null);
```

**Key additions**:

a. **`role="listbox"`** on the rail list container. Each `DraftRailItem` gets `role="option"`.

b. **`keydown` handler** on the list container:
```ts
function handleListKeydown(e: KeyboardEvent) {
  switch (e.key) {
    case 'ArrowDown': focusedIndex = Math.min(focusedIndex + 1, drafts.length - 1); scrollIntoView(); break;
    case 'ArrowUp': focusedIndex = Math.max(focusedIndex - 1, 0); scrollIntoView(); break;
    case 'Enter': if (drafts[focusedIndex]) onselect(drafts[focusedIndex].id); break;
    case 'n': case 'N': if (!e.metaKey && !e.ctrlKey) oncreate(); break;
    case 'Backspace': case 'Delete': handleArchiveFocused(); break;
    case 'd': case 'D': if (!e.metaKey && !e.ctrlKey) onduplicate?.(drafts[focusedIndex]?.id); break;
    case 'r': case 'R': if (tab === 'archive' && !e.metaKey && !e.ctrlKey) onrestore?.(drafts[focusedIndex]?.id); break;
    case '1': ontabchange('active'); break;
    case '2': ontabchange('scheduled'); break;
    case '3': ontabchange('posted'); break;
    case '4': ontabchange('archive'); break;
  }
}
```

c. **Scroll into view**: When `focusedIndex` changes, use `$effect` to call `element.scrollIntoView({ block: 'nearest' })` on the focused item.

d. **Focus management on tab switch**: Reset `focusedIndex` to 0 (or to the selected draft's index) when the tab changes.

e. **Undo toast**: Bottom bar in the rail that shows "Draft archived — Undo" for 5 seconds after an archive action.

f. **New props** (passed from shell):
```ts

onarchive: (id: number) => void;
onrestore: (id: number) => void;
```

g. **Tabs array update**: Add `{ key: 'posted', label: 'Posted' }`.

h. **Focus the rail list** on mount or when focus returns from the composer (via `Escape`). Expose a `focus()` method or use `bind:this`.
