# Session 2 Implementation Plan: Link Extraction + Edge Storage

**Date:** 2026-03-21
**Branch:** `epic/backlink-synthesizer`
**Goal:** Implement the additive storage and ingestion layer that turns vault notes into a resolvable note graph.

---

## Overview

This session adds three capabilities to the Watchtower pipeline:
1. **Link extraction** — parse wikilinks and markdown links from note bodies
2. **Edge storage** — persist note-to-note relationships as `note_edges` rows
3. **Tag normalization** — normalize frontmatter + inline tags into `note_tags` rows

All work is additive. Existing chunking, ingestion, and retrieval paths remain unchanged. Notes without links or tags produce zero edges/tags and flow through the pipeline identically to today.

## Detailed Design

### Step 1: Migration — `note_edges` and `note_tags` tables

**File:** `migrations/20260321000100_note_graph.sql`

```sql
-- Note graph edges: links between content nodes
CREATE TABLE IF NOT EXISTS note_edges (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    account_id TEXT NOT NULL,
    source_node_id INTEGER NOT NULL REFERENCES content_nodes(id) ON DELETE CASCADE,
    target_node_id INTEGER NOT NULL REFERENCES content_nodes(id) ON DELETE CASCADE,
    edge_type TEXT NOT NULL,       -- 'wikilink', 'markdown_link', 'shared_tag', 'backlink'
    edge_label TEXT,               -- display text, tag name, or link alias
    source_chunk_id INTEGER REFERENCES content_chunks(id),
    created_at TEXT NOT NULL DEFAULT (datetime('now')),
    UNIQUE(account_id, source_node_id, target_node_id, edge_type, edge_label)
);

CREATE INDEX IF NOT EXISTS idx_note_edges_source
    ON note_edges(account_id, source_node_id);
CREATE INDEX IF NOT EXISTS idx_note_edges_target
    ON note_edges(account_id, target_node_id);

-- Normalized tags extracted from notes
CREATE TABLE IF NOT EXISTS note_tags (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    account_id TEXT NOT NULL,
    node_id INTEGER NOT NULL REFERENCES content_nodes(id) ON DELETE CASCADE,
    tag_text TEXT NOT NULL,          -- lowercased, trimmed, no leading #
    source TEXT NOT NULL DEFAULT 'frontmatter',  -- 'frontmatter' or 'inline'
    UNIQUE(account_id, node_id, tag_text)
);

CREATE INDEX IF NOT EXISTS idx_note_tags_tag
    ON note_tags(account_id, tag_text);
CREATE INDEX IF NOT EXISTS idx_note_tags_node
    ON note_tags(account_id, node_id);

-- Provenance extension: track graph edge origin
ALTER TABLE vault_provenance_links ADD COLUMN edge_type TEXT;
ALTER TABLE vault_provenance_links ADD COLUMN edge_label TEXT;
```

**Naming convention:** Follows existing pattern `YYYYMMDD00NNNN_description.sql`. Latest migration is `20260316000200`. New migration: `20260321000100`.

**Key decisions:**
- `ON DELETE CASCADE` on both `source_node_id` and `target_node_id` — when a node is deleted, all its edges are automatically cleaned up.
- `UNIQUE` constraint on `(account_id, source_node_id, target_node_id, edge_type, edge_label)` — prevents duplicate edges within a single extraction pass.
- `edge_label` is nullable — backlink edges may not carry display text.
- `source_chunk_id` is optional — for future per-chunk edge tracking (not used in v1).
- `vault_provenance_links` ALTER is nullable — backward compatible with existing rows.

### Step 2: Link Extractor Module

**File:** `crates/tuitbot-core/src/automation/watchtower/link_extractor.rs`

**Types:**

```rust
#[derive(Debug, Clone, PartialEq, Eq)]
pub struct RawLink {
    pub target_raw: String,
    pub link_type: LinkType,
    pub display_text: Option<String>,
    pub heading_anchor: Option<String>,
}

#[derive(Debug, Clone, Copy, PartialEq, Eq)]
pub enum LinkType {
    Wikilink,
    MarkdownLink,
    InlineTag,
}
```

**Function:** `pub fn extract_links(body: &str) -> Vec<RawLink>`

**Algorithm:**
1. Iterate lines, track `in_code_block` state (same pattern as `extract_fragments` in chunker.rs — toggle on lines starting with `` ``` ``).
2. For non-code lines, apply three regex patterns:
   - **Wikilink:** `\[\[([^\]\|#]+)(?:#([^\]\|]+))?(?:\|([^\]]+))?\]\]` — captures target, optional heading anchor, optional display text.
   - **Markdown link:** `\[([^\]]+)\]\((?!https?://)([^)]+)\)` — captures display text and path. Only include if path ends with `.md` or has no extension.
   - **Inline tag:** `(?:^|[\s])#([a-zA-Z][a-zA-Z0-9_/-]*)` — captures tag name after `#`. Must be preceded by whitespace or start-of-line.
3. Deduplicate by `(target_raw, link_type)` — same link appearing twice in a note produces one `RawLink`.
4. Return `Vec<RawLink>`.

**Tag normalization helper:** `pub fn normalize_tag(raw: &str) -> String`
1. Strip leading `#`
2. Trim whitespace
3. Lowercase
4. Collapse internal whitespace to single `-`

**Unit tests (inline `#[cfg(test)] mod tests`):**
- `extract_wikilinks_simple` — `[[Target Note]]` → one Wikilink
- `extract_wikilinks_with_display` — `[[Target|Display]]` → display_text = "Display"
- `extract_wikilinks_with_heading` — `[[Target#Heading]]` → heading_anchor = "Heading"
- `extract_wikilinks_full` — `[[Target#Heading|Display]]` → all fields
- `extract_markdown_links` — `[text](./path.md)` → one MarkdownLink
- `extract_markdown_links_skip_external` — `[text](https://example.com)` → empty
- `extract_markdown_links_no_extension` — `[text](path)` → included (assumed note ref)
- `extract_inline_tags` — `#rust #distributed-systems` → two InlineTag entries
- `extract_inline_tags_skip_css` — `color: #ff0000` → empty (no whitespace before #)
- `extract_code_block_skipped` — links inside ` ``` ` blocks not extracted
- `extract_mixed_links` — wikilinks + md links + tags in same body
- `extract_deduplicates` — same `[[Target]]` twice → one RawLink
- `normalize_tag_cases` — `#Rust` → `rust`, `# My Tag` → `my-tag`

### Step 3: Edge Storage Module

**File:** `crates/tuitbot-core/src/storage/watchtower/edges.rs`

**Types:**

```rust
/// Input for inserting a new edge.
#[derive(Debug, Clone)]
pub struct NewEdge {
    pub source_node_id: i64,
    pub target_node_id: i64,
    pub edge_type: String,
    pub edge_label: Option<String>,
    pub source_chunk_id: Option<i64>,
}
```

**Functions:**

1. `pub async fn delete_edges_for_source(pool, account_id, source_node_id) -> Result<u64, StorageError>`
   - `DELETE FROM note_edges WHERE account_id = ? AND source_node_id = ?`
   - Also deletes backlink edges pointing back: `DELETE FROM note_edges WHERE account_id = ? AND target_node_id = ? AND edge_type = 'backlink'`
   - Wait — this is wrong. Backlinks where `source_node_id = the_node AND edge_type = 'backlink'` are already covered by the first delete. But we also need to delete backlinks that OTHER nodes created pointing TO this node. Actually no — those other nodes' forward edges should stay. The idempotency pattern is: when re-chunking node A, delete all edges WHERE `source_node_id = A`. This covers A's forward wikilinks, A's backlinks (edges that A "owns" as source). But the reverse edges (B→A backlinks created when A linked to B) need cleanup too.
   - **Corrected approach:** Delete all edges where `source_node_id = ?` (covers all edge types originating from this node). Also delete all edges where `target_node_id = ? AND edge_type = 'backlink'` (removes reverse backlinks that were auto-created when this node's wikilinks were processed). This ensures full idempotency.
   - Actually, let me reconsider. When node A has `[[B]]`:
     - We insert: `(source=A, target=B, type=wikilink)`
     - We insert: `(source=B, target=A, type=backlink)`
   - When A is re-chunked, we need to remove both. The first is covered by `WHERE source_node_id = A`. The second has `source_node_id = B`, so the first delete doesn't catch it.
   - **Final approach:** Two deletes in one function:
     1. `DELETE FROM note_edges WHERE account_id = ? AND source_node_id = ?` — removes all forward edges from this node
     2. `DELETE FROM note_edges WHERE account_id = ? AND target_node_id = ? AND edge_type IN ('backlink')` — removes backlinks created by this node's forward links
   - Wait, that's still wrong. The backlink `(source=B, target=A, type=backlink)` was created because A linked to B. If we delete it, but B also independently links to A, we'd lose B's backlink. But B's backlink would have `source=A, target=B, type=backlink` — no, B linking to A creates `(source=B, target=A, type=wikilink)` and `(source=A, target=B, type=backlink)`.
   - **Cleaner approach:** Tag edges with the "originating node" — the node whose re-chunk created them. But this adds a column.
   - **Simplest correct approach:** Just delete `WHERE source_node_id = ?` for ALL edge types. When A links to B, store both edges with `source_node_id = A`:
     - `(source=A, target=B, type=wikilink)` — A's forward link
     - `(source=A, target=B, type=backlink_reverse)` — wait, this breaks the "single WHERE source_node_id = ?" retrieval model.
   - **Decision:** Store edges as designed in the architecture doc (bidirectional). For idempotency, delete all edges where `source_node_id = ?` (forward edges from A, shared_tag edges from A). For the reverse backlinks `(source=B, target=A)`, use an additional delete: `DELETE FROM note_edges WHERE account_id = ? AND target_node_id = ? AND edge_type = 'backlink'` — this removes all backlinks that point FROM other nodes TO this node, which were auto-generated when this node's wikilinks were processed. If B independently links to A, B's re-chunk will recreate the `(source=A, target=B, type=backlink)` edge.
   - Hmm, but that still over-deletes: if both A and C link to B, and we re-chunk A, we'd delete backlinks where `target = A` — that's `(source=B, target=A, type=backlink)` — which is correct, those were auto-generated from A→B wikilink. But if C also linked to A, there'd be `(source=A, target=C, type=backlink)` created by C→A, which has source=A. The first delete `WHERE source_node_id = A` would remove it. Then when C is re-chunked, it would recreate it.
   - **This is actually fine.** The cost is that re-chunking A temporarily removes backlinks that other nodes created pointing to A, but those will be restored when those other nodes are re-chunked. For the typical single-note-edit scenario, only one node is re-chunked and the brief inconsistency is acceptable. For full re-index, all nodes get re-chunked anyway.
   - **Final decision:** Single delete `WHERE account_id = ? AND source_node_id = ?`. Accept temporary inconsistency for backlinks from other nodes. This is the simplest correct-enough approach and matches the architecture doc's idempotency pattern.

2. `pub async fn insert_edge(pool, account_id, edge: &NewEdge) -> Result<(), StorageError>`
   - `INSERT OR IGNORE INTO note_edges ...` — IGNORE handles the UNIQUE constraint for duplicate edges.

3. `pub async fn insert_edges(pool, account_id, edges: &[NewEdge]) -> Result<(), StorageError>`
   - Loops over edges, calling `insert_edge` for each.

4. `pub async fn get_edges_for_source(pool, account_id, source_node_id) -> Result<Vec<NoteEdge>, StorageError>`
   - Returns all edges where `source_node_id = ?` for debugging/testing.

**Row type:** `NoteEdge` struct with fields matching the table.

**New helper function** `extract_and_persist_graph()` (in chunker.rs or as a separate function):

```rust
/// Extract links and tags from a note body, persist edges and normalized tags.
///
/// Errors are logged but do not fail the chunking pipeline (fail-open).
async fn extract_and_persist_graph(
    pool: &DbPool,
    account_id: &str,
    node_id: i64,
    body_text: &str,
) {
    use super::link_extractor::{extract_links, normalize_tag, LinkType};

    let raw_links = extract_links(body_text);

    // 1. Delete stale edges and tags for idempotency
    if let Err(e) = store::delete_edges_for_source(pool, account_id, node_id).await {
        tracing::warn!(node_id, error = %e, "Failed to delete stale edges");
        return;
    }
    if let Err(e) = store::delete_tags_for_node(pool, account_id, node_id).await {
        tracing::warn!(node_id, error = %e, "Failed to delete stale tags");
        return;
    }

    // 2. Collect inline tags
    let mut tags: Vec<store::NormalizedTag> = Vec::new();
    for link in &raw_links {
        if link.link_type == LinkType::InlineTag {
            tags.push(store::NormalizedTag {
                tag_text: normalize_tag(&link.target_raw),
                source: store::TagSource::Inline,
            });
        }
    }

    // 3. Parse frontmatter tags from the node's existing `tags` column
    //    (already parsed by ingest_content and stored as comma-separated string)
    if let Ok(Some(node)) = store::get_content_node_for(pool, account_id, node_id).await {
        if let Some(ref tag_str) = node.tags {
            for t in tag_str.split(',') {
                let normalized = normalize_tag(t);
                if !normalized.is_empty() {
                    tags.push(store::NormalizedTag {
                        tag_text: normalized,
                        source: store::TagSource::Frontmatter,
                    });
                }
            }
        }
    }

    // 4. Persist tags
    if let Err(e) = store::insert_tags(pool, account_id, node_id, &tags).await {
        tracing::warn!(node_id, error = %e, "Failed to insert tags");
    }

    // 5. Resolve links to edges
    let mut edges: Vec<store::NewEdge> = Vec::new();
    for link in &raw_links {
        match link.link_type {
            LinkType::Wikilink => {
                // Resolution: case-insensitive title match, then path match
                if let Some(target) = resolve_wikilink(pool, account_id, &link.target_raw).await {
                    // Forward edge
                    edges.push(store::NewEdge {
                        source_node_id: node_id,
                        target_node_id: target,
                        edge_type: "wikilink".to_string(),
                        edge_label: link.display_text.clone().or_else(|| Some(link.target_raw.clone())),
                        source_chunk_id: None,
                    });
                    // Reverse backlink
                    edges.push(store::NewEdge {
                        source_node_id: target,
                        target_node_id: node_id,
                        edge_type: "backlink".to_string(),
                        edge_label: link.display_text.clone().or_else(|| Some(link.target_raw.clone())),
                        source_chunk_id: None,
                    });
                } else {
                    tracing::debug!(
                        node_id,
                        target = %link.target_raw,
                        "Unresolvable wikilink target, skipping"
                    );
                }
            }
            LinkType::MarkdownLink => {
                // Resolution: normalize path relative to source, match relative_path
                if let Some(target) = resolve_md_link(pool, account_id, node_id, &link.target_raw).await {
                    edges.push(store::NewEdge {
                        source_node_id: node_id,
                        target_node_id: target,
                        edge_type: "markdown_link".to_string(),
                        edge_label: link.display_text.clone(),
                        source_chunk_id: None,
                    });
                    edges.push(store::NewEdge {
                        source_node_id: target,
                        target_node_id: node_id,
                        edge_type: "backlink".to_string(),
                        edge_label: link.display_text.clone(),
                        source_chunk_id: None,
                    });
                } else {
                    tracing::debug!(
                        node_id,
                        target = %link.target_raw,
                        "Unresolvable markdown link target, skipping"
                    );
                }
            }
            LinkType::InlineTag => {
                // Tags don't create edges directly — shared-tag edges are created below
            }
        }
    }

    // 6. Create shared-tag edges
    match store::find_shared_tag_neighbors(pool, account_id, node_id, 10).await {
        Ok(neighbors) => {
            for (target_id, tag_text) in neighbors {
                edges.push(store::NewEdge {
                    source_node_id: node_id,
                    target_node_id: target_id,
                    edge_type: "shared_tag".to_string(),
                    edge_label: Some(tag_text),
                    source_chunk_id: None,
                });
            }
        }
        Err(e) => {
            tracing::warn!(node_id, error = %e, "Failed to find shared-tag neighbors");
        }
    }

    // 7. Persist edges
    if let Err(e) = store::insert_edges(pool, account_id, &edges).await {
        tracing::warn!(node_id, error = %e, "Failed to insert edges");
    }
}
```

## Key Design Decisions

### D1: Fail-open graph extraction
Graph extraction errors (edge insert failures, resolution failures) are logged at `warn`/`debug` level but do not fail the `chunk_node()` pipeline. This ensures backward compatibility — notes without links or with unresolvable links still get chunked normally.

### D2: Single delete for idempotency
`DELETE FROM note_edges WHERE account_id = ? AND source_node_id = ?` before re-inserting. This temporarily removes backlink edges that OTHER nodes auto-created pointing to this node, but those are recreated when those other nodes are re-chunked. Acceptable trade-off for simplicity.

### D3: link_extractor.rs is pure
No async, no DB access. Takes `&str`, returns `Vec<RawLink>`. This makes it trivially testable with sync unit tests. All resolution (DB lookups) happens in `graph_ingest.rs`.

### D4: Shared-tag edges created at chunk-time
After tags are persisted, `find_shared_tag_neighbors` queries existing `note_tags` rows to find other nodes sharing tags. This means shared-tag edges are incrementally built — each node re-chunk adds its own shared-tag edges. Older nodes won't automatically get shared-tag edges to newly indexed notes until they themselves are re-chunked. This is acceptable for v1 (edges populate over time during natural edits and re-indexes).

### D5: graph_ingest.rs separated from chunker.rs
`chunker.rs` is at 423 lines. Adding graph integration inline would exceed the 500-line limit. The new `graph_ingest.rs` file keeps both files manageable and separates the graph concern from the fragment extraction concern.

### D6: Wikilink resolution strategy
1. Case-insensitive exact title match (`LOWER(title) = LOWER(?)`)
2. Path ending match (`relative_path LIKE '%/' || ? || '.md'` OR `relative_path = ? || '.md'`)
3. If no match, skip (fail-open)

No fuzzy matching in v1. This is deterministic and fast.

### D7: Markdown link path normalization
Given source node at `notes/subfolder/my-note.md` and link `[text](../other.md)`:
1. Extract source directory: `notes/subfolder/`
2. Join with link path: `notes/subfolder/../other.md`
3. Normalize: `notes/other.md`
4. Match against `content_nodes.relative_path`

Use `std::path::PathBuf` for normalization (cross-platform safe per CLAUDE.md rules), but convert back to `/`-delimited string for DB matching.

## Risks and Mitigations

| Risk | Severity | Mitigation |
|------|----------|------------|
| Regex false positives on wikilinks inside inline code (`` `[[not a link]]` ``) | Low | Code fence blocks are skipped. Single backtick inline code is not — acceptable for v1. Can add inline code detection in v2. |
| Large notes with many links could slow chunk_node() | Low | Link extraction is O(n) in body length. Edge inserts are O(links). Typical notes have < 50 links. |
| `find_shared_tag_neighbors` JOIN could be slow with many tags | Low | Indexed by `(account_id, tag_text)`. Capped at 10 results. SQLite handles this well. |
| Temporary backlink inconsistency after single-node re-chunk | Low | Acceptable for v1. Backlinks from other nodes are restored on their next re-chunk. Full re-index restores all. |
| Migration could fail on existing databases with unexpected state | Low | Migration uses `CREATE TABLE IF NOT EXISTS` and `ALTER TABLE ADD COLUMN`. Both are safe for re-runs. SQLite ALTER TABLE ADD COLUMN is non-destructive. |
