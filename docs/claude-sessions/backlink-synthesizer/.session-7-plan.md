- **Phase 1 (Internal)**: Enable for team accounts; validate funnel events appear in tracing logs
- **Phase 2 (Beta)**: Enable for opted-in users via synthesis toggle default=ON
- **Phase 3 (GA)**: Enable for all users with vault sources
- **Kill switch**: Synthesis toggle OFF disables all graph features without code change
- **Monitoring**: Watch for errors in graph_expansion, telemetry endpoint latency, empty-graph rate

## Key Design Decisions

1. **backlinkFunnel tests use vi.mock** — The functions are thin wrappers around `trackFunnel`. Testing them verifies correct event names and property shapes, not funnel infrastructure. This is the right granularity.

2. **No new E2E test for full backlink flow** — The E2E tests require a running server + database + Obsidian plugin mock. The existing unit + integration tests at each layer provide sufficient coverage. The E2E gap is documented in the QA matrix as an accepted risk.

3. **Server integration tests seed data directly** — Consistent with existing `graph_neighbor_tests.rs` pattern. No new test helpers needed; reuse `build_test_state()` and `seed_graph_data()`.

4. **Consistency fixes are minimal** — If discrepancies are found between docs and code, fix trivial ones (typos, missing fields). Flag architectural discrepancies in release-readiness.md as residual risks rather than blocking.

5. **Go/no-go is per-subsystem** — The release readiness doc evaluates each layer independently (ingestion, retrieval, UX, provenance, privacy, backward compat, fallback, analytics) so a partial rollout is possible if one area is no-go.

## Risks and Mitigations

| Risk | Impact | Mitigation |
|------|--------|------------|
| Existing tests fail after Session 6 changes | Blocks progress | Fix immediately in Step 1; Session 6 handoff says all green |
| Tag-only neighbor test reveals query bug | Medium | Fix the query in graph_expansion.rs or watchtower/tags.rs |
| Consistency pass finds doc/code divergence | Low-Medium | Document in release-readiness; fix if trivial |
| backlinkFunnel mock setup complex | Low | Functions are simple wrappers; vi.mock is standard |
| E2E tests require server running | Low | Skip E2E for graph features; document as accepted gap |
