# Session 13 — Validation and Launch Readiness Plan

## Objective

Validate the full Obsidian Vault to Post Loop initiative end to end, fix any regressions, produce a QA matrix and release-readiness report, and ensure all roadmap documentation is internally consistent.

**This session produces no source code changes** (unless quality gates reveal regressions). All deliverables are documentation.

---

## Order of Operations

### Phase 1: Quality Gates (fix regressions if any)

**Step 1.1 — Run all four quality gates sequentially**

```bash
cargo fmt --all && cargo fmt --all --check
RUSTFLAGS="-D warnings" cargo test --workspace
cargo clippy --workspace -- -D warnings
npm --prefix dashboard run check
```

- If any gate fails, triage the failure:
  - **In-scope for this epic** (vault, chunks, provenance, assist, watchtower, vault UI) → fix immediately.
  - **Out-of-scope** (unrelated pre-existing issue) → document in release-readiness as known issue, not a blocker.
- Re-run gates after any fix until all four pass cleanly.

**Step 1.2 — Verify Tauri build compiles**

```bash
cd dashboard && npm run build
```

- This confirms the frontend production bundle is valid (SPA mode).
- Tauri native build (`npm run tauri build`) is out of scope for CI but note if there are known issues.

## Key Design Decisions

1. **No new source code** — Session 13 is validation and documentation only. The feature set is frozen at Session 12.
2. **Code-reading QA, not runtime QA** — Since this is an automated session without a running server, manual coverage is done by reading code paths, not clicking through UI. Runtime E2E testing is an operator responsibility pre-launch.
3. **Deferred ≠ blocked** — Features listed as "future" in handoffs are explicitly not release blockers. The report must clearly distinguish shipped from deferred.
4. **ComposeWorkspace tech debt accepted** — 960 lines is 2.4x the 400-line limit but doesn't block release. Document as P2 tech debt with a remediation plan.
5. **GO recommendation expected** — Based on Session 12 handoff showing all gates passing and no blocking issues, the recommendation should be GO unless Phase 1 reveals regressions.
