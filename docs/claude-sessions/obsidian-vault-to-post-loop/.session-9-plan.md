# Session 9 — Composer "From Vault" Workflow — Implementation Plan

## Goal

Add a "From Vault" flow to the composer inspector so users can search notes, pick fragments, generate content grounded in vault material, see citation chips, and preserve provenance refs through draft creation. The existing "From Notes" (free-text paste) flow remains available as a tab/toggle peer.

---

## Design Decisions

### D1: Tab toggle between "From Notes" and "From Vault" (not radio buttons)

The UX blueprint suggests an Auto/From Vault radio pair. However the current codebase has a `showFromNotes` boolean that opens FromNotesPanel. Rather than replacing it, add a peer panel (FromVaultPanel) and use a two-segment toggle inside InspectorContent to switch between them. This keeps the existing From Notes path fully intact and lets vault search live beside it.

- `showFromNotes` becomes `notesPanelMode: 'notes' | 'vault' | null` (null = closed).
- Backward-compatible: `onopenotes` sets mode to `'notes'`, a new `onopenvault` sets it to `'vault'`.

### D2: FromVaultPanel as a new sibling component (not modifying FromNotesPanel)

FromNotesPanel is 324 lines and self-contained. Vault search/select is a different interaction (search → expand note → pick chunks). A new `FromVaultPanel.svelte` avoids bloating the notes panel and keeps separation of concerns.

### D3: Citation chips as a new CitationChips component rendered in the canvas area

After generation from vault, citations appear as small pills below the editor (in the canvas body snippet, before the undo banner). This keeps them visible while composing but doesn't clutter the text area itself. The component is reusable for future reply citation display.

### D4: Provenance flows through undoSnapshot and buildComposeRequest

- After generation, `vault_citations` from the assist response are stored in a new `vaultCitations` state variable.
- `buildComposeRequest` gains an optional `provenance` field populated from the current citations.
- Undo restores the previous citations (or clears them).

### D5: Max 3 fragment selections (per UX blueprint)

The vault search panel enforces a max of 3 selected chunks. Checkboxes disable when limit is reached.

### D6: Selected node IDs flow through existing assist API

The Session 8 API already supports `selected_node_ids` on assist endpoints. When generating from vault, we pass the node_ids of notes containing the selected chunks. The backend handles retrieval.

Actually — the API accepts `selected_node_ids` (node-level), but the UX allows chunk-level selection. Since the backend's `build_draft_context_with_selection` retrieves all chunks for selected nodes, we pass the unique node_ids of the selected chunks. This is sufficient for Session 9. Chunk-level selection refinement is a future enhancement.

## Files to Create

### 1. `dashboard/src/lib/components/composer/FromVaultPanel.svelte` (NEW, ~280 lines)

Vault search and fragment selection panel for the inspector.

**Structure:**
```
<script lang="ts">
  Props: mode, hasExistingContent, ongenerate, onclose, onundo, showUndo, selectedCitations (bindable)
  State: searchQuery, notes (VaultNoteItem[]), expandedNote (VaultNoteDetail | null),
         selectedChunks (Set<number> of chunk_ids), loading, error, debounceTimer

  Functions:
  - searchNotes(q): calls api.vault.searchNotes({ q, limit: 20 }), debounced 300ms
  - expandNote(id): calls api.vault.noteDetail(id), shows chunks
  - toggleChunk(chunk): add/remove from selectedChunks (max 3 enforced)
  - handleGenerate(): builds selectedNodeIds from selectedChunks, calls ongenerate
  - onMount: load initial notes (no query = recent notes)
</script>

Template:
- Header row: "From Vault" label + close button
- Search input with magnifying glass icon
- Note list (scrollable, max-height ~200px):
  - Each note: title (or filename), chunk_count badge, expand chevron
  - Expanded note: list of chunks with checkboxes
    - Chunk: heading_path, snippet preview, checkbox
    - Checkbox disabled when 3 already selected and this one isn't
- Selection counter: "N of 3 fragments selected"
- Generate button (disabled when 0 selected)
- Replace confirmation (reuses same pattern as FromNotesPanel)
- Undo button (same pattern)
- Error and empty states
```

**Keyboard access:**
- Search input auto-focuses on mount
- Tab cycles through notes and chunks
- Enter on a note expands it
- Space on a chunk checkbox toggles it

**Mobile:**
- Touch targets ≥44px (pointer: coarse media query)
- Search input font-size: 16px to prevent iOS zoom

### 2. `dashboard/src/lib/components/composer/CitationChips.svelte` (NEW, ~120 lines)

**Styling:**
- Small pills with subtle background (color-mix accent 8%)
- Flex-wrap row so they flow naturally

```ts
async function handleGenerateFromVault(selectedNodeIds: number[]) {
  submitError = null;
  undoSnapshot = { mode, text: tweetText, blocks: [...threadBlocks], citations: [...vaultCitations] };
  undoMessage = 'Content replaced.';

  if (mode === 'thread') {
    const result = await api.assist.thread(topicWithCue(voiceCue, tweetText || 'Generate thread'), selectedNodeIds);
    threadBlocks = result.tweets.map((text, i) => ({
      id: crypto.randomUUID(), text, media_paths: [], order: i
    }));
    if (result.vault_citations) vaultCitations = result.vault_citations;
  } else {
    const context = voiceCue || 'Generate a tweet from vault context';
    const result = await api.assist.tweet(topicWithCue(voiceCue, tweetText || 'Generate tweet'), selectedNodeIds);
    tweetText = result.content;
    if (result.vault_citations) vaultCitations = result.vault_citations;
  }
  voicePanelRef?.saveCueToHistory();
  notesPanelMode = null;
  showUndo = true;
  if (undoTimer) clearTimeout(undoTimer);
  undoTimer = setTimeout(() => { showUndo = false; }, 10000);
}
```

## API Usage Map

| UI Action | API Call | Notes |
|-----------|----------|-------|
| Open vault panel | `api.vault.searchNotes({})` | Load recent notes on mount |
| Type in search | `api.vault.searchNotes({ q })` | Debounced 300ms |
| Expand note | `api.vault.noteDetail(id)` | Load chunks for selection |
| Generate tweet | `api.assist.tweet(topic, nodeIds)` | Returns `vault_citations` |
| Generate thread | `api.assist.thread(topic, nodeIds)` | Returns `vault_citations` |
| Submit/schedule | `buildComposeRequest(...)` | `provenance` field from citations |

## Risks and Mitigations

| Risk | Likelihood | Impact | Mitigation |
|------|-----------|--------|------------|
| ComposeWorkspace exceeds 400-line Svelte limit | Already exceeded (893 lines) | Low | Pre-existing. Not introduced by this session. Note in handoff. |
| `notesPanelMode` refactor breaks existing From Notes flow | Medium | Medium | Keep FromNotesPanel untouched. Gate with `=== 'notes'`. Test both paths. |
| Vault search returns empty when no sources configured | Expected | Low | Show "No vault sources configured" empty state with link to settings. |
| Debounced search feels sluggish | Low | Low | 300ms debounce is standard. Show loading indicator. |
| Mobile drawer too tall with vault panel content | Low | Medium | Panel content is scrollable. Max-height on note list (200px). |
| `cmd+shift+v` conflicts with paste-as-plain-text | High | Medium | Use a different shortcut. Check collisions. Consider `cmd+shift+u` for "Use vault" or just palette-only without keyboard shortcut. |
| Fragment selection limited to node-level API | Known | Low | Session 8 API accepts `selected_node_ids`. Chunk-level filtering is backend enhancement for later. |
