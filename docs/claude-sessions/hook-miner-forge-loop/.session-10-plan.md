# Session 10 Implementation Plan — Instrumentation & Success Metrics

**Date:** 2026-03-22
**Session:** 10 of 11
**Goal:** Instrument Hook Miner and Forge so adoption, fallback usage, and sync reliability are measurable after release.

---

## Current State Assessment

### What exists
- **Frontend funnel system:** `funnel.ts` provides `trackFunnel()` + `FunnelEvent` interface. `backlinkFunnel.ts` has 13 typed helpers, all prefixed `backlink.`. Optional `bufferEvent()`/`flushToBackend()` relay to backend.
- **Backend telemetry intake:** `POST /api/telemetry/events` in `telemetry.rs`. Validates batch size (max 50), enforces `backlink.` prefix, logs via `tracing::info!`. 4 tests.
- **No Hook Miner or Forge events exist** — the `backlink.angles_mined` and `backlink.angle_fallback` helpers in `backlinkFunnel.ts` cover the graph synthesis flow, not the Hook Miner angle-mining feature or Forge sync.

### Key constraint
The backend prefix check (`EVENT_PREFIX = "backlink."`) must be extended to also accept `hook_miner.` and `forge.` prefixes without breaking existing `backlink.*` events.

---

## Design Decisions

| ID | Decision | Rationale |
|----|----------|-----------|
| D71 | Use an allowlist of prefixes instead of a single `EVENT_PREFIX` | Enables `hook_miner.*` and `forge.*` namespaces alongside `backlink.*`. Extensible for future feature namespaces. |
| D72 | Create a new `hookMinerFunnel.ts` file (not extend `backlinkFunnel.ts`) | Backlink funnel tracks the graph synthesis flow. Hook Miner and Forge are distinct features with different event schemas. Separation keeps each file focused and under size limits. |
| D73 | Properties never include raw note content, file contents, or frontmatter values | Privacy-safe: only structural metadata (source_path basename, session_id, angle_kind, boolean flags, error reason strings). Source path is the filename stem only, not full path. |
| D74 | Backend allowlist is a `const` array checked with `.iter().any()` | Simple, testable, no config needed. Adding a new namespace is a one-line change. |
| D75 | Telemetry remains logging-only (no DB writes) | Matches existing design. Structured logs can be aggregated by external tools (Loki, CloudWatch, etc.). |
| D76 | Events use snake_case with dot-separated namespace | Matches existing `backlink.*` convention exactly. |

## Implementation Steps (Exact Order)

### Step 1: Extend backend telemetry allowlist

**File:** `crates/tuitbot-server/src/routes/telemetry.rs`

**Changes:**
1. Replace `const EVENT_PREFIX: &str = "backlink."` with:
   ```rust
   /// Allowed event name prefixes (namespace isolation).
   const ALLOWED_PREFIXES: &[&str] = &["backlink.", "hook_miner.", "forge."];
   ```
2. Update the validation loop (lines 48-57) from:
   ```rust
   if !ev.event.starts_with(EVENT_PREFIX) {
   ```
   to:
   ```rust
   if !ALLOWED_PREFIXES.iter().any(|p| ev.event.starts_with(p)) {
   ```
3. Update the error message to list all allowed prefixes:
   ```rust
   format!(
       "Invalid event name \"{}\": must start with one of {:?}",
       ev.event, ALLOWED_PREFIXES
   )
   ```
4. Add new tests:
   - `hook_miner_prefix_accepted` — batch with `hook_miner.angles_shown` returns 204
   - `forge_prefix_accepted` — batch with `forge.sync_succeeded` returns 204
   - `mixed_namespace_batch_accepted` — batch with `backlink.*`, `hook_miner.*`, and `forge.*` events returns 204
   - `unknown_prefix_rejected` — batch with `other.event` returns 400
   - Existing tests remain unchanged

**Rationale:** This is the foundation — backend must accept new events before frontend can relay them.

### Step 2: Create Hook Miner + Forge funnel helpers

**File:** `dashboard/src/lib/analytics/hookMinerFunnel.ts`

**Event definitions:**

#### Hook Miner Events

```typescript
// 1. Angle cards shown to user
export function trackAnglesShown(
    angleCount: number,
    sessionId: string,
    sourcePathStem: string,     // filename without extension, no full path
    localEligible: boolean      // whether source is local filesystem
): void

// 2. User selects an angle card
export function trackAngleSelected(
    angleKind: string,          // e.g. "contradiction", "data_point", "aha_moment"
    sessionId: string,
    sourcePathStem: string,
    evidenceCount: number       // number of evidence items on the card
): void

// 3. Fallback triggered (weak signal or no neighbors)
export function trackFallbackOpened(
    reason: string,             // "weak_signal" | "no_neighbors" | "synthesis_disabled" | "api_error" | "timeout"
    sessionId: string,
    acceptedCount: number       // how many neighbors were accepted (0 for no_neighbors)
): void
```

#### Forge Events

```typescript
// 4. Analytics sync consent prompt shown on Activity page
export function trackForgePromptShown(
    sourcePathStem: string,
    localEligible: boolean
): void

// 5. User enables analytics sync (from prompt or settings)
export function trackForgeEnabled(
    sourcePathStem: string,
    enabledFrom: string         // "prompt" | "settings"
): void

// 6. Forge sync completed successfully
export function trackForgeSyncSucceeded(
    tweetsSynced: number,
    threadsSynced: number,
    entriesNotFound: number,
    filesNotFound: number
): void

// 7. Forge sync failed
export function trackForgeSyncFailed(
    reason: string,             // error category, not raw error message
    stage: string               // "db_query" | "file_write" | "percentile_calc"
): void
```

**Implementation pattern:** Each helper calls `trackFunnel()` from `./funnel.ts` with the appropriate namespace prefix. Properties are explicitly typed — no raw note content, no full paths, no frontmatter values.

**Privacy safeguards:**
- `sourcePathStem` is explicitly documented as filename stem only (e.g., `"my-note"`, not `"/Users/alice/vault/my-note.md"`)
- No `content`, `body`, `frontmatter`, or `raw_text` properties on any event
- Error reasons are categorical strings, not stack traces

## Risks and Mitigations

| Risk | Severity | Mitigation |
|------|----------|------------|
| Prefix allowlist change could break existing events | Low | Existing `backlink.*` prefix remains in the array. All existing tests kept and must pass. New tests added for new prefixes. |
| `sourcePathStem` could accidentally receive full paths | Medium | Add a `sanitizePathStem()` utility that strips directory components. Test that it rejects paths with separators. Document the contract clearly. |
| Forge sync events need to be emitted from backend (Rust), not frontend | Medium | `forge.sync_succeeded` and `forge.sync_failed` should be emitted from the frontend after the WebSocket notification or API response, not from Rust. The Rust side uses `tracing::info!` for server-side observability. Frontend events track UX-visible outcomes. |
| Event properties could drift from documentation | Low | Tests pin the exact event names and property keys. The instrumentation-plan.md serves as the source of truth. |
| No DB storage means events are lost if logs aren't aggregated | Low | Acceptable for v1 — structured logging is the established pattern. Future sessions can add persistence if needed. |
